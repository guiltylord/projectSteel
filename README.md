# projectSteel

Автоматизация заполнения веб-формы с прохождением графической капчи.

## Описание

Скрипт автоматически заполняет форму на целевом сайте, используя данные из конфигурации, и проходит проверку капчи с помощью OCR (ddddocr).

## Требования

- **Python 3.10** (обязательно)
- Chromium (устанавливается автоматически через Playwright)

## Установка

### 1. Создайте виртуальное окружение

```bash
python -m venv .venv310
```

### 2. Активируйте виртуальное окружение

**Windows:**
```bash
.venv310\Scripts\activate
```

**Linux/macOS:**
```bash
source .venv310/bin/activate
```

### 3. Установите зависимости

```bash
pip install -r requirements.txt
```

### 4. Установите браузер для Playwright

```bash
playwright install chromium
```

## Настройка

### 1. Создайте файл `.env`

Скопируйте пример конфигурации:

```bash
copy .env.example .env
```

### 2. Отредактируйте `.env`

Заполните файл своими данными:

- **Техническая конфигурация (C0–C11):** селекторы элементов страницы и URL целевого сайта
- **Данные пользователя (D1–D5):** персональные данные для заполнения формы

Пример:
```env
C0=https://target-site.com/search
C1=input[name="field_1"]
C2=input[name="field_2"]
C3=input[name="field_3"]
C4=#dropdown_id
C5=#date_input_id
C6=button#submit_search
C7=#captcha_input_field
C8=#captcha_image_id
C9=10
C10=5
C11=#captcha_submit_button

D1=Ivanov
D2=Ivan
D3=Ivanovich
D4=01.01.1990
D5=77
```

## Запуск

После настройки запустите скрипт:

```bash
python main.py
```

## Результат работы

- При успешном прохождении формы результат сохраняется в файл `capture_result.html`
- Логи процесса выводятся в консоль

## Структура проекта

```
projectSteel/
├── main.py              # Основной скрипт
├── requirements.txt     # Зависимости Python
├── .env.example         # Пример конфигурации
├── .env                 # Актуальная конфигурация (создаётся вручную)
└── README.md            # Документация
```

## Примечания

- Скрипт выполняет до 5 попыток распознавания капчи (настраивается в `C10`)
- Для работы требуется стабильное интернет-соединение
- Браузер запускается в обычном режиме (не headless) для отладки
